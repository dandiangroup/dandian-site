# Сайт Dandian → RSS → Дзен

Простой статический сайт-блог. Дзен забирает статьи из ленты `rss.xml` и публикует их у тебя на канале как родные публикации платформы. Лента собрана строго по техтребованиям Дзена.

```
dandian-site/
├── articles/      ← исходники статей (.md). СЮДА кладёшь новые статьи
├── assets/        ← обложки (.jpg). Сюда кладёшь картинку статьи
├── config.json    ← настройки сайта (адрес, название) — заполнить 1 раз
├── build.py       ← генератор (Python, без зависимостей)
├── robots.txt     ← доступ роботу Дзена (Mail.ru) уже открыт
└── dist/          ← РЕЗУЛЬТАТ сборки (его и публикует хостинг)
```

---

## Шаг 1. Куда выложить (бесплатный хостинг)

Рекомендую вариант A — после настройки сайт сам пересобирается, тебе не нужен Python на компьютере.

### Вариант A — GitHub + Cloudflare Pages (авто-сборка, рекомендую)
1. Создай бесплатный аккаунт на github.com и новый репозиторий, загрузи туда всю папку `dandian-site` (кнопка **Add file → Upload files**, можно перетащить).
2. Зайди на dash.cloudflare.com → **Workers & Pages → Create → Pages → Connect to Git**, выбери репозиторий.
3. В настройках сборки укажи:
   - **Build command:** `python3 build.py`
   - **Build output directory:** `dist`
4. Деплой создаст адрес вида `https://dandian-xxxx.pages.dev`.
5. Впиши этот адрес в `config.json` → поле `base_url`, сохрани и снова залей файл в GitHub. Сайт пересоберётся сам.

(Netlify работает так же: build command `python3 build.py`, publish directory `dist`.)

### Вариант B — без Git, перетаскиванием
1. Установи Python 3 (python.org).
2. Впиши адрес будущего сайта в `config.json` → `base_url`.
3. Запусти в папке: `python3 build.py`
4. Папку `dist/` целиком перетащи на **app.netlify.com/drop** (Netlify Drop) — получишь адрес сразу.

> После смены `base_url` всегда пересобирай (`build.py`) — адреса в ленте должны совпадать с реальным доменом.

---

## Шаг 2. Подключить ленту к Дзену
1. Зайди в Студию Дзена под своим каналом.
2. Раздел **«Дзен для медиа» / подключение RSS** → добавь адрес ленты: `https://ТВОЙ-АДРЕС/rss.xml`
3. Дождись проверки. Робот Дзена скачивает файл раз в несколько минут; статьи появляются после индексации.

Требования, которые уже соблюдены в этом сайте: лента доступна по HTTPS, весит < 10 МБ, робот Mail.ru допущен через `robots.txt`, обложки больше 800×400, даты в нужном формате. От тебя — чтобы сам сайт соответствовал правилам Дзена к качеству и был доступен.

---

## Шаг 3. Как добавлять новые статьи (еженедельный поток)
1. Положи новый файл `articles/НАЗВАНИЕ.md` (шаблон — см. любую существующую статью; вверху блок `--- ... ---` с полями `title, slug, date, category, cover, description`).
2. Положи обложку в `assets/` и впиши её имя в поле `cover`.
3. Залей файлы в репозиторий (вариант A — сайт пересоберётся сам) **или** запусти `build.py` и перезалей `dist/` (вариант B).
4. Лента `rss.xml` обновится автоматически, Дзен подтянет новые статьи.

Важно про `date`: ставь **реальное время публикации**. Дзен считает свежими статьи за последние 8 дней — старее не индексирует.

## Про автоматизацию загрузки статей (ответ на вопрос «можешь сам заливать?»)
Я не могу в фоне сам заливать файлы на твой хостинг — у меня нет постоянного процесса и доступов. Но цепочка после попадания статьи в репозиторий — полностью автоматическая: загрузка файла → пересборка на хостинге → обновление `rss.xml` → Дзен забирает. Каждую неделю я отдаю готовый комплект `.md` + обложки, ты кладёшь их в репозиторий одним перетаскиванием.

Если захочешь убрать и этот шаг — следующий уровень: подключаем папку Google Drive как источник и ставим GitHub Action по расписанию, который сам забирает оттуда новые статьи и коммитит. Тогда мне достаточно складывать файлы в Drive, а сайт обновляется без твоего участия. Скажи — настроим.

---

## Честные риски
RSS-импорт — официальный путь, но Дзен строго относится к качеству и может не пропустить канал на массовом ИИ-контенте без «прогрева». Публикуй ровным темпом, держи статьи живыми и полезными — так канал не уйдёт в спам-фильтр.
